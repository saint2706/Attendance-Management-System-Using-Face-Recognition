## üìù Documentation Standards

### Writing Guidelines
- Use clear, concise language
- Include code examples that can be copy-pasted
- Test all commands before documenting them
- Keep technical accuracy over marketing speak
- Use inclusive language (see [Code of Conduct](../CODE_OF_CONDUCT.md))

### Updating Documentation
When you update code that affects documentation:

1. **Update relevant docs in the same PR**
2. **Test all commands and examples**
3. **Update screenshots if UI changed** (`make docs-screenshots`)
4. **Check for broken links**
5. **Update the CHANGELOG.md**

### Documentation Issues
Found a problem in the docs? Please:
- [Open an issue](https://github.com/saint2706/Attendance-Management-System-Using-Face-Recognition/issues/new/choose)
- Use the "Documentation" issue template
- Include the specific file and section
- Suggest improvements if possible
